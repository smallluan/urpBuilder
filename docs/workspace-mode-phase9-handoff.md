# Phase9 个人/团队空间分离改造（前后端协作说明）

## 1. 本次目标
- 将工作台从“个人资源 + 团队资源同页筛选”切换为“个人空间 / 团队空间”二元模式。
- 切换空间后，构建页面与构建组件自动按当前空间取数，不再展示 scope 筛选。
- 团队页改为“团队看板”语义（仅团队空间展示有意义内容）。

## 2. 前端已完成行为
- 新增 `workspaceMode`：`personal | team`，并本地持久化。
- 侧边栏空间切换器支持：
  - 切换到个人空间。
  - 选择团队后切换到团队空间。
  - 在 popup 内触发“创建团队”。
- `BuildPage` / `BuildComponent`：
  - 取消手动 scope 选择。
  - `personal` 模式自动使用 `mine=true`。
  - `team` 模式自动使用 `ownerType=team & ownerTeamId=currentTeamId`。
- 团队页：
  - 移除“我的团队”左侧区域。
  - 仅保留当前团队看板（团队资产、成员等）。
  - 当处于个人空间时给出切换提示。
- 路由移除个人信息页入口（`/profile` 不再挂载到主路由）。
- 第二轮收口：
  - 清理 `resourceFilters` 中遗留的 `scope/scopeTouched` 字段。
  - 数据管理页（`DataApi` / `DataConstance`）接入空间语义：个人空间提示切换，团队空间按当前团队展示。
  - 彻底删除个人信息页面代码文件。
- 第三轮收口：
  - 保存弹窗移除“个人/团队”手动归属选择，改为由当前 `workspaceMode` 自动决定保存归属。

## 3. 本次后端是否必须配合
当前阶段（含第二轮收口）**不依赖后端改动即可落地**：
- 现有列表接口已支持 `mine` / `ownerType` / `ownerTeamId` 参数，足以支撑空间分离。
- 团队看板数据可复用现有团队详情与团队资产聚合接口。
- 数据管理页面当前接入为空间语义占位，无新增接口依赖。

## 4. 建议后端后续配合（非阻塞）
以下均为后续优化项，不影响当前版本交付：
### 4.1 统一空间参数（建议）
建议后续新增统一参数（例如 `workspaceMode`），减少前端在不同接口上拼装参数差异：
- `workspaceMode=personal` => 等价 `mine=true`
- `workspaceMode=team` + `workspaceTeamId`

### 4.2 数据管理接口提前对齐
数据管理页面已接入空间语义占位，但后续接口建议直接按空间语义设计：
- 默认按当前空间返回结果。
- 避免再引入“个人/团队混合后再筛选”的模型。

### 4.3 权限错误语义
团队空间下若 `workspaceTeamId` 无效或无访问权限，建议明确返回：
- 404（团队不存在）
- 403（无权访问团队）
前端可按状态码给出更明确提示。

### 4.4 保存归属校验对齐（建议）
虽然前端已按 `workspaceMode` 固化保存归属，建议后端也做同口径校验，避免绕过前端造成脏数据：
- 团队空间保存时：`ownerType` 必须为 `team`，且 `ownerTeamId` 必须为当前团队。
- 个人空间保存时：`ownerType` 必须为 `user`，且不允许写入其他团队归属。
- 若归属与上下文不一致，建议返回 `400`（参数不合法）或 `403`（无权操作）。

## 5. 联调检查清单
- 个人空间：构建页仅返回个人资源。
- 团队空间：构建页仅返回当前团队资源。
- 切换团队后：构建页与团队看板同步切换。
- 无团队场景：可从空间 popup 创建团队并进入团队看板。
- 权限场景：成员/管理员/拥有者在团队资源操作按钮行为符合约束。
