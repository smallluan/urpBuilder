## 1. 素材管理：网格卡片被纵向拉满

| 文件 | `src/pages/DataAssets/style.less` |
|------|-----------------------------------|

**现象**：`.data-assets-grid` 在纵向 flex 里占满剩余高度；CSS Grid 默认 `align-content: stretch` / `align-items: stretch`，单行时行轨道与单元格被拉高，卡片跟着铺满，缩略图挤在顶部。

**必须保留的样式**（在 `.data-assets-grid` 上）：

```less
align-content: start;
align-items: start;
```

**勿回退**：不要删掉这两行，也不要改回仅依赖默认 stretch，否则素材卡片会再次出现「整列超高、内容贴顶」。

## 2. 数据常量 API：兼容 snake_case、JSON 文本与 CRUD

| 文件 | `src/api/dataConstant.ts` |
|------|---------------------------|

**要点**：

- **`coerceApiDataConstantRecord`**：统一网关/旧接口的 `snake_case` 与 `value_json` 等字段，避免前端拿不到 `valueType` / `value`。

## 3. 常量管理页：编辑、删除、预览与表单陷阱

| 文件 | `src/pages/DataConstance/index.tsx`、`style.less` |
|------|---------------------------------------------------|

**要点**：

- 支持 **创建 / 编辑**（`drawerMode`、`editingId`）、**删除**（确认框）、**预览**（完整 JSON）。
- **`useLayoutEffect`**：编辑打开时从 `records` 再灌一层 `draft`，解决部分环境下首帧 batch 后表单不同步。
- **禁止使用 `Form.FormItem` 包裹未配合 Form `name` 的受控 Input/Select**：TDesign 会用内部 `formValue` 覆盖子组件 `value`，导致编辑时名称/类型不回显；已改为自定义 `.data-constance-page__field` 布局（见注释）。
- JSON 校验错误文案强调须为 **标准 JSON**（双引号等），与 `JSON.parse` 一致。
- `CodeMirrorEditor` 使用 `key` 与 `drawerFormKey`/`editingId` 避免切换记录时状态粘连。

**勿回退**：不要把抽屉表单改回「仅用 Form.FormItem + 受控组件」而不处理 Form 覆写问题。

## 4. 动态列表项：禁止单独删除 / 剪切 / 清空子节点

| 文件 | `src/constants/componentBuilder.ts`、`ComponentAsideLeft.tsx`、`ComponentBody.tsx` |
|------|-------------------------------------------------------------------------------------|

**要点**：

- 新增 **`isDynamicListItemNode`**：`type === 'DynamicList.Item'`。
- 删除、剪切、清空子节点时 **跳过或拦截** 该节点，并 **`MessagePlugin.warning`**；右键菜单 **`canCut` / `canDelete`** 对列表项为 false。
- 画布与左侧结构树逻辑应对齐，避免一侧仍可破坏行模板壳。

**勿回退**：不要移除这些 guard，否则用户可能删掉必须的 `DynamicList.Item`，结构损坏。

## 5. 列表动态绑定：`__listBinding` 多字段映射

| 文件 | `ComponentConfigPanel.tsx`、`propAccessors.tsx`、`PreviewRenderer.tsx` |
|------|-----------------------------------------------------------------------|

**数据形状**：

- 新：``__listBinding.value`` 支持 **`{ mappings: [{ prop, field }, ...] }`**。
- 兼容旧：单组 **`{ prop, field }`** 仍通过 `normalizeListBindingMappings` 转为数组。

**配置面板**：

- **可绑定属性** 来自当前组件 **`mergedEditableProps`**（非 `__` 开头），不再使用写死的 `LIST_BINDABLE_PROP_OPTIONS`。
- **动态列表** 下字段选项来自 **常量数据源**：`dataSourceConfig` 指向常量、且常量为 **`array` + 元素为同构对象**；否则展示明确 **error**（空数组、非对象元素、行间 key 不一致等）。
- **`loadDataSourceResources`** 与相关 **`useEffect` 须在 `if (!activeNode) return` 之前声明**，否则会违反 Hooks 规则（React #310）。

**运行时**：

- `applyListBindingToNode` 对 **多组 mapping** 逐组 `getListFieldRawValue` 并写入对应 prop。

**勿回退**：

- 三处 `normalizeListBindingMappings` / `applyListBindingToNode` 逻辑应保持一致（构建器 accessor 与预览）。
- 勿把 `loadDataSourceResources` 挪回 `activeNode` 早返回之后。
